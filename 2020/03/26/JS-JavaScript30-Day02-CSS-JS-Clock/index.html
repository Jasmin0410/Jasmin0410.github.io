【JS】 JavaScript30－Day02 CSS + JS Clock

發表於 2020-03-26

作品：Demo
第二天挑戰要做出純 CSS + JS 的時鐘

目標

  1. 指針轉動(CSS－transform)
  2. 取得時間，依時間呈現指針位置

筆記

CSS－transform

一開始會發現指針們……
哎呀～怎麼沒有從 12:00 開始阿！

先來試試 CSS 的 transform
找到 .hand 加上 transform: rotate(90deg)
transform 語法是能讓元件進行轉動、平移、縮放、傾斜的效果
在這裡使用到的 rotate 就是轉動的功用，90deg 就是所要轉動的角度值

CSS－transform-origin

但加了上去…….
怎麼發現這轉動的原點怎麼怪怪的

是轉動了但感覺很怪0.0
這就得提到 transform 中心點－ transform-origin(x,y) 的問題了！
transform-origin 預設值為 (50% ,50%)，也就是元件的正中央
這個概念以正方形的情況會比較好理解
依下圖，可理解為ｘ軸的中心 (50%) 跟ｙ的中心 (50%)

(PS.努力畫中心點….用我拙劣的小畫家XDD)
因此就是transform-origin(50%,50%)，導致了在我們指針的結果
要讓指針以正確的起始點去旋轉
所以我們將 .hand 加上 transform-origin: 100%，改變起始點

順利完成～指針調整到 12:00

取得時間

上半部了解了轉動跟起始點
接下來就是要讓指針可以依目前時間來轉動
首先取得當前時間，並分別抓出時、分、秒的數值

  • new Date()：取得當前時間
  • .getHours()：取得時數
  • .getMinutes()：取得分數
  • .getSeconds()：取得秒數

接下來計算時針、分針、秒針的轉動角度

  • 時針：(360 / 12 ) deg * hour + ((minute / 60) * 30)
    因為不想讓時針變動太突然，所以依據分鐘加上了每小時間移動角度
  • 分針：(360 / 60) deg * minute
  • 秒針：(360 / 60) deg * second

依時間轉動指針

計算完角度後，直接用 JS 操作 style 去讓指針轉動

不過你一定會發現時間怪怪的！！
明明加上的角度是對的，但跑出來的就不對
別忘記了！
我們一開始讓指針轉 90 度，回歸原點的步驟
所以將旋轉角度改了，同時得加上初始設定的 90 度唷！

加上定時器

最後就加上定時器做每秒更新

耶~完成!
